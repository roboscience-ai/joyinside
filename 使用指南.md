# JoyInside 机器人语音接入 — 使用指南

本文档面向团队成员，说明如何使用本仓库将 **JoyInside 平台的 Token、ASR（语音识别）、TTS（语音合成）** 接入自有机器人。

> **定位说明**：本项目**只使用** JoyInside 的鉴权、ASR、TTS 能力，**不使用**其智能体对话、技能、指令控制等能力。对话逻辑由我们自己的程序负责。

---

## 一、整体架构

```
┌─────────────┐     Token (REST)      ┌──────────────────┐
│  你的机器人  │ ────────────────────> │ api.joyinside.com │
└─────────────┘                       └──────────────────┘
       │
       │  WebSocket（ASR / TTS）
       ▼
┌──────────────────┐
│ ws.joyinside.com │
└──────────────────┘
       │
       ├── 上行：PCM 音频  →  ASR 识别文本
       └── 下行：TTS 音频  ←  文本合成请求
```

**典型流程**：

1. 用 Access Key / Secret Key 换取短期 `accessToken`
2. 建立 WebSocket 连接
3. **ASR**：上传麦克风 PCM → 收到识别文本 → 交给自己的 LLM / 业务逻辑
4. **TTS**：发送合成请求 → 收到 PCM 音频 → 扬声器播放

## 二、前置条件

### 2.1 JoyInside 控制台配置

在 [https://joy-inside.jd.com](https://joy-inside.jd.com) 完成以下步骤（只需做一次）：

| 步骤 | 位置 | 产出 |
|------|------|------|
| 1. 注册并审核企业 | 登录 → 提交企业信息 | `vendorId`（企业 ID） |
| 2. 创建产品型号 | 资源管理 → 型号库 | 产品型号 |
| 3. 创建应用 | 应用管理 → 新建应用 | `appId`（应用 ID） |
| 4. 获取 API 密钥 | 开发者中心 → API 密钥 | `Access Key`、`Secret Key` |
| 5. 注册设备 | 设备管理 → 新建，或运行本仓库注册脚本 | `botId`（设备 ID） |

**应用创建时注意**：

- 接入类型选择 **「语音智能体接入」**（需要音频链路）
- 人设、技能等可保持最简配置，本项目会忽略智能体自动回复

### 2.2 本地环境

- Python 3.10 及以上
- 可访问 `api.joyinside.com` 和 `ws.joyinside.com` 的网络

## 四、快速开始

### 4.1 安装依赖

```powershell
cd joyinside_robot
python -m pip install -r requirements.txt
```

### 4.2 填写配置

```powershell
copy .env.example .env
```

编辑 `.env` 文件：

```ini
JOYINSIDE_ACCESS_KEY=你的AccessKey
JOYINSIDE_SECRET_KEY=你的SecretKey
JOYINSIDE_VENDOR_ID=你的企业ID
JOYINSIDE_APP_ID=你的应用ID
JOYINSIDE_BOT_ID=你的设备botId
JOYINSIDE_DEVICE_ID=robot-sn-001
```

**各字段获取位置**：

| 环境变量 | 获取方式 |
|----------|----------|
| `JOYINSIDE_ACCESS_KEY` | 开发者中心 → API 密钥 |
| `JOYINSIDE_SECRET_KEY` | 开发者中心 → API 密钥 |
| `JOYINSIDE_VENDOR_ID` | 个人信息 → 企业信息 → 企业 ID |
| `JOYINSIDE_APP_ID` | 应用管理 → 应用卡片下方 |
| `JOYINSIDE_BOT_ID` | 设备管理 → 设备 ID 列；或通过注册脚本获取 |
| `JOYINSIDE_DEVICE_ID` | 自定义设备 SN，仅首次 API 注册时使用 |

> ⚠️ **安全提醒**：`.env` 含 Secret Key，**不要提交到 Git**。建议将 `.env` 加入 `.gitignore`。

### 4.3 注册设备（若控制台尚未创建设备）

```powershell
python examples/register_device.py
```

成功后终端会输出 `botId`，将其写入 `.env` 的 `JOYINSIDE_BOT_ID`。

若已在控制台手动创建设备，可跳过此步，直接把设备 ID 填入 `JOYINSIDE_BOT_ID`。

### 4.4 测试 TTS（语音合成）

```powershell
python examples/tts_demo.py "你好，我是机器人助手"
```

成功后会在项目根目录生成 `output_tts.pcm`。

使用 ffplay 播放（需安装 FFmpeg）：

```powershell
ffplay -f s16le -ar 16000 -ac 1 output_tts.pcm
```

### 4.5 测试 ASR（语音识别）

准备一段 **16kHz、16bit、单声道** 的 PCM 音频文件，例如 `test_input.pcm`：

```powershell
python examples/asr_demo.py test_input.pcm
```

终端会输出识别到的文本。

**用 FFmpeg 将 WAV/MP3 转为所需 PCM 格式**：

```powershell
ffmpeg -i input.wav -ar 16000 -ac 1 -f s16le test_input.pcm
```

### 4.6 完整流程演示

```powershell
python robot_demo.py
```

演示 ASR 识别 → 简单回复逻辑 → TTS 合成的完整链路。若根目录存在 `test_input.pcm` 会走 ASR；否则仅做 TTS 演示。

## 五、集成到自己的机器人

### 5.1 最简代码示例

```python
from config import JoyInsideConfig
from joyinside import JoyInsideAuth
from joyinside.speech import JoyInsideSpeech, SyncSpeechHelper
from joyinside.audio import chunk_pcm, read_pcm_file

# 加载配置
cfg = JoyInsideConfig.from_env()
auth = JoyInsideAuth(cfg.access_key, cfg.secret_key)

# 创建语音客户端
speech = JoyInsideSpeech(
    bot_id=cfg.bot_id,
    get_token=lambda: auth.get_token(bot_id=cfg.bot_id),
)
speech.connect()
helper = SyncSpeechHelper(speech)

# --- ASR：麦克风 PCM → 文本 ---
pcm_data = read_pcm_file("test_input.pcm")  # 替换为你的麦克风采集逻辑
user_text = helper.recognize_and_wait(chunk_pcm(pcm_data))
print("用户说:", user_text)

# --- 你的业务逻辑 ---
reply = your_llm_or_rules(user_text)  # 替换成自己的大脑

# --- TTS：文本 → PCM 音频 ---
audio_pcm = helper.speak_and_collect(reply)
# 将 audio_pcm 送入你的扬声器播放模块

speech.close()
```

### 5.2 回调模式（适合实时流式播放）

```python
def on_asr(text: str):
    print("识别完成:", text)
    # 触发你的对话逻辑...

def on_tts_audio(data: bytes, meta: dict):
    player.enqueue(data)  # 边收边播

speech = JoyInsideSpeech(
    bot_id=cfg.bot_id,
    get_token=lambda: auth.get_token(bot_id=cfg.bot_id),
    on_asr_final=on_asr,
    on_tts_audio=on_tts_audio,
)
speech.connect()
speech.speak("你好")
```

### 5.3 核心 API 说明

| 类 / 方法 | 作用 |
|-----------|------|
| `JoyInsideAuth.get_token()` | 获取 accessToken，内置缓存，过期前 5 分钟自动刷新 |
| `JoyInsideSpeech.connect()` | 建立 WebSocket 连接 |
| `JoyInsideSpeech.speak(text)` | 纯 TTS，**不触发** JoyInside 智能体 |
| `JoyInsideSpeech.recognize_pcm(chunks)` | 上传 PCM 帧列表进行 ASR |
| `JoyInsideSpeech.interrupt()` | 打断当前输出（ASR 后自动调用，阻止智能体跟读） |
| `SyncSpeechHelper.speak_and_collect(text)` | 同步阻塞，返回完整 TTS 音频 bytes |
| `SyncSpeechHelper.recognize_and_wait(chunks)` | 同步阻塞，返回 ASR 识别文本 |

## 六、生产部署：Token 中转服务

**问题**：机器人固件或端侧程序不应保存 `Secret Key`。

**方案**：在服务端运行 `token_service.py`，机器人只从中转服务获取短期 Token。

### 6.1 启动服务

```powershell
python token_service.py
```

服务默认监听 `http://127.0.0.1:8765`。

### 6.2 机器人侧调用

```http
GET http://你的服务器:8765/token
```

响应示例：

```json
{
  "access_token": "eyJ...",
  "bot_id": "0428f41d89fc436b8d63a529cf7141a7"
}
```

机器人拿到 `access_token` 后，用它建立 WebSocket 连接即可。

## 七、音频格式要求

### 上行（ASR 输入）

| 参数 | 值 |
|------|-----|
| 格式 | PCM（默认） |
| 采样率 | 16000 Hz（也支持 24000 / 32000） |
| 位深 | 16 bit |
| 声道 | 单声道 |
| 每帧时长 | ≤ 300 ms（默认 120 ms） |
| 发送节奏 | 每帧间隔 = 帧时长（否则延迟异常） |

### 下行（TTS 输出）

| 参数 | 值 |
|------|-----|
| 默认格式 | PCM 16kHz 16bit 单声道 |
| 可选格式 | Opus（需先调用 `update_audio_config()`） |

### 切换 Opus 下行（可选）

```python
speech.connect()
speech.update_audio_config(
    output_codec="opus",
    output_frame_size_ms="40",
)
```

需等待收到 `SERVER_VOICE_CHAT_UPDATED` 事件后再发起 TTS。

## 八、工作原理说明

### 8.1 为什么需要 WebSocket？

JoyInside 的 ASR 和 TTS **没有独立的 REST 接口**，统一走 WebSocket 语音通道：

- **TTS**：发送 `CLIENT_INPUT_TEXT_TO_SPEECH` 事件，平台明确说明**不会触发智能体**
- **ASR**：流式上传 `AUDIO` 帧，平台返回 `contentType: ASR` 消息

### 8.2 如何屏蔽智能体自动回复？

上传音频后，平台默认还会走 Agent + TTS 流程。本项目通过以下方式屏蔽：

1. 使用**手动模式**（`needManualCall=true`，默认开启）
2. ASR 收到最终结果后，自动发送 `CLIENT_INTERRUPT` 打断智能体
3. 忽略后续 `AGENT` 类型的下行消息

### 8.3 Token 有效期

- `accessToken` 有效期 **2 小时**
- `JoyInsideAuth` 会在过期前 5 分钟自动刷新
- 若 Token 失效，WebSocket 建连会失败，需重新获取

## 九、常见问题

### Q1：WebSocket 连接失败

| 可能原因 | 解决办法 |
|----------|----------|
| Token 过期 | 重新调用 `get_token()` |
| botId 错误 | 检查 `.env` 中 `JOYINSIDE_BOT_ID` |
| 时间戳偏差 | 确保本机时间与网络时间误差 < 15 分钟 |
| 网络不通 | 检查能否访问 `ws.joyinside.com` |

### Q2：报错 `REPEAT_CLIENT_SESSION`

同一 `botId` **只能有一个** WebSocket 连接。检查是否有其他程序或测试脚本占用了连接。

### Q3：ASR 识别结果为空或超时

- 确认音频为 16kHz / 16bit / 单声道 PCM
- 确认音频时长足够（建议 > 1 秒）
- 确认发送节奏正确（每帧后 sleep 对应时长）
- 检查是否收到了 `CLIENT_AUDIO_FINISH`（手动模式必须发送）

### Q4：TTS 没有声音 / 文件为空

- 确认 WebSocket 已成功连接
- 等待足够时间（`SyncSpeechHelper` 默认超时 60 秒）
- 检查下行消息中是否有 `contentType: TTS`

### Q5：智能体还是说话了

- 确认 `auto_interrupt_agent=True`（默认开启）
- 收到 ASR 结果后是否成功发送了 `CLIENT_INTERRUPT`
- 控制台应用可尽量减少技能配置

### Q6：长期记忆混乱

- 不要在所有设备上使用相同的固定 `uid`
- 应传真实用户标识，或留空不传

## 十、安全规范

1. **Secret Key 只放服务端**，不要写入机器人固件或前端代码

3. 生产环境使用 `token_service.py` 或自建 Token 中转，机器人只拿短期 Token
4. 定期轮换 Access Key / Secret Key
